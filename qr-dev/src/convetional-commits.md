# Conventional Commits

Conventional Commits is a convention for standardizing commit messages in software projects, as proposed in this [specification](https://www.conventionalcommits.org/pt-br/v1.0.0/). It defines a clear structure for commits, making them more readable and enabling automation (e.g., changelog generation and semantic versioning).

For a quick reference, you can check this [cheatsheet](https://gist.github.com/qoomon/5dfcdf8eec66a051ecd85625518cfd13), which summarizes the most common commit types and formats.

Additionally, tools like [commitlint](https://commitlint.js.org/) help enforce these conventions. It’s a CLI tool that allows teams to share configurations and integrate commit message validation into their development workflow, including CI pipelines.
